## Overview
Use API keys to authenticate requests to the API. Keep keys secret and rotate them if they are exposed.

## Authentication methods
### API key (recommended)
Send your key in the `Authorization` header:

- `Authorization: Bearer <API_KEY>`

> Never put API keys in client-side apps (mobile/web). Use them only on trusted servers.

## Get an API key
1. Create an account in the dashboard.
2. Go to **Settings → API Keys**.
3. Create a key for the environment you need:
- **Test** for development
- **Live** for production

# Authentication

## POST /v1/auth/signup
Create a tenant + admin user.

**Body**
```json
{
  "companyName": "Acme Travel",
  "email": "ops@acme.com",
  "password": "supersecret123"
}
```

**Response**
```json
{
  "token": "<jwt>",
  "tenant": { "id": "...", "name": "Acme Travel" },
  "apiKey": "tb_..."
}
```

## POST /v1/auth/login
Sign in with email + password.

**Body**
```json
{ "email": "ops@acme.com", "password": "supersecret123" }
```

**Response**
```json
{ "token": "<jwt>", "tenant": { "id": "...", "name": "Acme Travel" } }
```

## GET /v1/auth/me
Get current tenant + user.

## GET /v1/auth/api-keys
List API keys (masked).

## POST /v1/auth/api-keys/rotate
Rotate API key; returns a **new** key. Only the latest key is active.
